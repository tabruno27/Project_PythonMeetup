# Meetup Telegram Bot
Многофункциональный Telegram-бот для сопровождения митапов, конференций и других публичных мероприятий. 

Бот помогает автоматизировать взаимодействие между организаторами, спикерами и слушателями: позволяет управлять расписанием, задавать вопросы из зала в реальном времени, управлять статусами докладов и делать массовые рассылки.

## Основные возможности (Ролевая система)

Система автоматически подстраивает меню и доступные команды в зависимости от роли пользователя (Организатор, Спикер, Слушатель).

### Для слушателей (Пользователей)
- **`/scheduler`** — Просмотр актуального расписания всех докладов.
- **`/active`** — Информация о текущем (активном) докладе.
- **`/ask [вопрос]`** — Задать вопрос спикеру прямо во время его выступления.
- **Донаты** — Кнопка для поддержки проекта (интеграция с CloudTips).

### Для спикеров
- *Включает все функции слушателя, а также:*
- **Уведомления** — Автоматическое получение сообщения при добавлении в расписание.
- **`/my_questions`** — Просмотр всех вопросов от слушателей к своим докладам (с пометками об ответе).
- **`/start_talk` / `/end_talk`** — Управление статусом своего выступления (открывает и закрывает возможность задавать вопросы из зала).

### Для организаторов
- *Включает все функции, а также:*
- **`/add_speaker`** — Пошаговое добавление нового спикера и его доклада (через FSM: запрос ID, ФИО, темы, времени начала и окончания).
- **`/delete_speaker`** — Удаление спикера и всех связанных с ним докладов.
- **`/mass_sms`** — Массовая рассылка сообщений всем зарегистрированным участникам мероприятия.
- **`/update_schedule`** — Обновление расписания докладов.

## Технологический стек

- **Язык:** Python 3.10+
- **Фреймворк:** [Aiogram 3.x](https://docs.aiogram.dev/) (Асинхронный Telegram API)
- **База данных:** SQLite (через кастомный ORM/DB manager в модуле `datacenter`)
- **Архитектура:** FSM (Finite State Machine) для сложных диалогов, динамическое обновление Reply-клавиатур и меню команд (`BotCommandScopeChat`).

## Установка и запуск

1. Настройка виртуального окружения и установка зависимостей

```bash
python -m venv venv
source venv/bin/activate  # Для Linux/macOS
venv\Scripts\activate     # Для Windows
pip install -r requirements.txt
```

2. Настройка переменных окружения

```bash
TG_TOKEN=ваш_токен_бота_от_BotFather
ORGANIZER_TELEGRAM_ID=ваш_личный_telegram_id
```

Примечание: ORGANIZER_TELEGRAM_ID (только цифры) нужен для выдачи полных прав администратора при использовании команд управления спикерами и рассылками.

3. Запуск бота
```bash
python bot.py
```
